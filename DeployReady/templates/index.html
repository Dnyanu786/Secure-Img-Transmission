Security Upgrade: Zero Password Transmission

Problem solved: Sharing passwords over WhatsApp or Email sends them as plaintext — anyone who intercepts the message can decrypt your image. This system eliminates that risk using QR Code + TOTP (Time-based One-Time Password). The password never leaves the server. The receiver scans a QR code and enters a 6-digit code that expires in 30 seconds.

Security Architecture

🔐

AES-256-CBC + PBKDF2

Your password is never used directly as a key. PBKDF2-HMAC-SHA256 with 200,000 iterations and a random 32-byte salt derives both the encryption key and HMAC key — making brute-force attacks computationally infeasible.

📲

QR + TOTP Key Exchange

The password stays on the server — always. The QR code carries only a session token and a TOTP seed. The 6-digit OTP expires every 30 seconds. Even if the QR is intercepted, it's useless after 30s.

🧮

Encrypt-then-MAC

HMAC-SHA256 is applied over the IV + ciphertext using a separate MAC key (not the encryption key). Any tampering with the encrypted file is detected and decryption is rejected before any cipher operation runs.
